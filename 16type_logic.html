AI 활용 유형 진단 로직 가이드

1단계: 진단 시작 및 질문 제시

  • 진단 프로세스는 4개의 '진단 축'을 평가하기 위한 질문들로 구성됩니다.
  • 각 축마다 3개의 질문을 제시하여 사용자의 성향을 정확하게 파악합니다.

진행 방식: 각 축(이해, 방식, 스타일, 인식)에 해당하는 질문 3개를 순서대로 사용자에게 제시하며, 상반된 두 개의 선택지(A/B) 중 하나를 선택하게 합니다. 선택된 답변은 각 축의 성향으로 기록됩니다.

이해 (Comprehension): C (개념) vs A (응용)

무엇을 측정하는가? AI를 학습하고 접근할 때, 근본적인 '원리'에 집중하는지, 아니면 '실용적인 활용법'에 집중하는지를 판단합니다.

C (Concept - 개념 중심)

"AI가 어떻게 작동하지?", "왜 이런 결과가 나올까?"처럼 내부 동작 원리, 기본 개념, 기술적 배경에 대한 호기심이 강한 유형입니다.

A (Application - 응용 중심)

"이걸로 뭘 할 수 있지?", "어떻게 하면 내 숙제를 더 빨리 끝낼 수 있을까?"처럼 AI를 도구로써 실제 문제 해결에 즉시 적용하려는 성향이 강한 유형입니다.

방식 (Approach): S (안정) vs E (탐험)

무엇을 측정하는가? AI를 활용할 때, 검증되고 안정적인 방식을 선호하는지, 아니면 새롭고 실험적인 방식을 시도하는 것을 즐기는지를 판단합니다.

S (Stable - 안정형)

이미 알려진 효과적인 프롬프트, 예측 가능한 결과를 내는 안정적인 기능 사용을 선호합니다. 실패의 위험을 최소화하려는 성향입니다.

E (Exploratory - 탐험형)

AI의 새로운 기능, 예상치 못한 답변, 알려지지 않은 활용법을 찾아내는 것을 즐깁니다. 시행착오를 두려워하지 않는 성향입니다.

스타일 (Style): S (구조) vs I (직관)

무엇을 측정하는가? AI에게 명령을 내리고 소통할 때, 체계적이고 명확한 계획을 따르는지, 아니면 즉흥적이고 유연한 대화를 선호하는지를 판단합니다.

S (Structured - 구조형)

단계별 지시, 명확한 역할 부여, 표나 목록과 같은 구조화된 결과물을 요청하는 것을 선호합니다. 계획적이고 체계적인 접근을 합니다.

I (Intuitive - 직관형)

친구와 대화하듯 자유롭게 질문을 던지고, 추상적인 아이디어를 탐구하며, AI의 반응에 따라 대화의 흐름을 바꿔나가는 것을 선호합니다.

인식 (Perception): C (신중) vs O (낙관)

무엇을 측정하는가? AI의 결과물과 기술 자체를 받아들일 때, 비판적이고 조심스러운 태도를 보이는지, 아니면 긍정적이고 수용적인 태도를 보이는지를 판단합니다.

C (Cautious - 신중형)

AI의 답변이 틀릴 수 있다는 '환각(Hallucination)' 현상을 경계하고, 항상 정보의 출처를 확인하며, 잠재적인 위험성을 먼저 고려하는 태도를 보입니다.

O (Optimistic - 낙관형)

AI가 가져올 긍정적인 변화와 가능성에 더 주목하며, AI의 답변을 신뢰하고 적극적으로 활용하려는 태도를 보입니다.

2단계: 결과 도출 로직

사용자가 12개 질문에 대한 답변을 마치면, 시스템은 아래 로직에 따라 최종 유형 코드를 생성합니다.

축별 성향 집계 (예시)

  • 이해 축: C를 선택한 개수와 A를 선택한 개수를 비교합니다.
    • C > A 이면, 첫 번째 코드는 C가 됩니다.
    • A > C 이면, 첫 번째 코드는 A가 됩니다.

최종 유형 코드 조합

  • 위의 과정에서 결정된 4개의 알파벳 코드를 순서대로 조합하여 최종 유형 코드를 생성합니다.
  • 예시: 사용자가 [이해 축='A', 방식 축='E', 스타일 축='S', 인식 축='C']로 판별되었다면, 최종 유형 코드는 AESC가 됩니다.
